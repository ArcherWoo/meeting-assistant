# 2026-04-07 产品口径补充：Know-how 权限与会话隐私

## 1. Chat 下 Know-how 的权限口径

普通用户通过 Chat 调起 Know-how 时，只能使用自己有权访问的规则。

当前产品口径中，“有权访问”包括：

- 自己的个人规则
- 自己所在用户组的共享规则
- 显式授权给自己的规则
- 显式授权给自己所在用户组的规则
- `public` 规则

管理员在 Chat 中可以使用全部 Know-how 规则。

当用户询问“规则库里有多少条规则”“有哪些分类”这类问题时，系统返回的也必须是当前用户可见范围，不是全库总量。

## 2. Know-how 管理口径

管理员：

- 可管理全量规则
- 可管理全局分类
- 可全量导入导出

组内 Know-how manager：

- 可创建自己的个人规则
- 可创建共享给本组的 baseline 规则
- 只能管理本组共享规则
- 只能管理本组共享规则正在使用的分类
- 只能追加导入、导出本组规则

普通用户：

- 可使用自己有权限访问的规则
- 不具备规则库级管理能力

## 3. 会话隐私口径

会话默认是个人隐私数据，不因为用户是管理员或组内 manager 就获得额外可见性。

当前统一口径：

- 管理员只看自己的会话
- 组内 manager 只看自己的会话
- 普通用户只看自己的会话

产品上不再保留“管理员查看所有人会话”的默认行为。

## 4. 设计原则

- Know-how 是“按权限注入”的增强资源，不是“全局公开的系统说明书”
- 会话是“用户个人工作流状态”，不是“默认可审计的后台流水”
- 管理权限与使用权限必须分开

具体来说：

- 能用某条规则，不代表能编辑、删除、导入导出、管理分类
- 能管理本组规则，不代表能查看所有人的会话
